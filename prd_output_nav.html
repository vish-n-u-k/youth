Executive Summary

Young Engineers Enrollment & Operations Platform

A lightweight, flexible enrollment and communication system built for children's enrichment programs. Eliminates manual work, reduces enrollment friction, and increases conversions from lead to trial to paid enrollment. Supports offline payment methods (Zelle) and automates communication around the enrollment lifecycle.

Open Questions

IDQuestionAreaPriorityStatusDefaultResolution
OQ-01Should trial classes be free, or can some programs charge for trials? If charged, should the trial fee be deducted from the program fee upon enrollment?mediumTrials are free by default. Admin can optionally set a trial fee per program. Trial fee is not automatically deducted from enrollment fee.
OQ-02When enrollment closes (capacity reached or window closed), should parents be able to join a waitlist?mediumNo waitlist in MVP. Parents see a 'Program Full' message. Admin can manually override to add enrollments beyond capacity.
OQ-03Should there be an enrollment approval step (admin reviews and approves each enrollment) or is enrollment instant upon form submission?highEnrollment is instant upon form submission. Admin can cancel/modify enrollments after the fact.
OQ-04For the 'Trial Attended' status — how is this tracked? Does admin manually mark it, or is there an attendance check-in feature?mediumAdmin manually marks trial as attended in the lead detail view. Automated attendance tracking deferred to Phase 2.
OQ-05Should the system support sibling discounts or family-level pricing rules?mediumNo automatic sibling discount in MVP. Admin can manually apply a coupon or override pricing per enrollment to achieve the same effect.

Out of Scope

  • Payment gateway integration (Stripe, card payments) — Phase 2
  • SMS and WhatsApp communication channels — Phase 2
  • Parent portal with persistent login and enrollment history — Phase 2
  • Instructor attendance tracking — Phase 2
  • Reporting exports (CSV, PDF) — Phase 2
  • Multi-location management UI — Phase 2 (data model supports it from MVP)
  • Curriculum or lesson plan management
  • Student progress tracking or grading
  • Inventory or materials management
  • Financial accounting or invoicing beyond payment status tracking
  • Mobile native app (responsive web is the MVP approach)
