Requirements (5)

PT-REQ-01unknown

Dashboard lists all enrollments with columns: parent name, child name, program, amount due, amount received, payment status, payment method, last updated

Acceptance Criteria

All columns are visible and sortable; list can be filtered by payment status (pending, paid, partial, waived)

PT-REQ-02unknown

Admin can set payment status to: pending, paid, partial, waived

Acceptance Criteria

Status change is saved immediately and reflected across all views

PT-REQ-03unknown

Admin can record: amount received, payment method, and free-text notes (e.g., 'Paid via Zelle on Jan 12')

Acceptance Criteria

All fields are saved and visible on the payment detail screen

PT-REQ-04unknown

Payment detail shows the full price breakdown from the enrollment (base fee, registration, coupon, total)

Acceptance Criteria

Price breakdown matches the enrollment record and is read-only on the payment screen

PT-REQ-05unknown

Page displays: Zelle recipient name/email/phone, exact amount to pay, and a reference note to include (e.g., child name + program name)

Acceptance Criteria

All payment instruction fields are configured by admin in settings and displayed accurately to the parent
