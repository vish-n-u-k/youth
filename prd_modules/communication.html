Requirements (6)

CM-REQ-01unknown

Admin can create email templates with: subject line, body (rich text), and variable placeholders (e.g., {{parent_name}}, {{program_name}}, {{trial_date}}, {{payment_amount}})

Acceptance Criteria

Templates can be saved with placeholders; preview mode shows sample data substituted into placeholders

CM-REQ-02unknown

System provides default templates for: lead received, trial confirmation, trial reminder, enrollment confirmation, payment reminder, class start reminder

Acceptance Criteria

All 6 default templates exist on initial setup; admin can edit but not delete defaults

CM-REQ-03unknown

Admin can configure automation rules mapping triggers to templates: lead created → lead received email, trial scheduled → trial confirmation, trial date approaching → trial reminder, enrollment completed → enrollment confirmation + payment instructions, payment overdue → payment reminder, class start approaching → class start reminder

Acceptance Criteria

Each trigger has an assigned template; admin can enable/disable individual automations; disabled automations do not send emails

CM-REQ-04unknown

Admin can configure timing for time-based triggers: trial reminder (N hours before), payment overdue (N days after enrollment), class start reminder (N days before)

Acceptance Criteria

Timing is configurable per trigger; default values are pre-set (24h for trial reminder, 3 days for payment overdue, 2 days for class start)

CM-REQ-05unknown

Log displays: recipient email, subject, template used, trigger event, sent timestamp, delivery status (sent, failed)

Acceptance Criteria

All automated emails appear in the log; log is filterable by recipient, template, and status

CM-REQ-06unknown

Failed emails are flagged and admin is notified for manual follow-up

Acceptance Criteria

Failed emails show 'Failed' status with error reason; admin sees a notification or badge for unsent communications
