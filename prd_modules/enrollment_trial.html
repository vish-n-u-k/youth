Requirements (12)

ET-REQ-01unknown

Page displays all active programs as cards or list items showing: program name, age group, schedule summary, price, and whether trial is available

Acceptance Criteria

All active programs appear; closed/draft programs do not; each card has a 'Book Trial' and/or 'Enroll Now' action button

ET-REQ-02unknown

Programs at capacity show a 'Full' indicator and disable the enroll button

Acceptance Criteria

When enrollment count equals capacity, the 'Enroll Now' button is disabled and 'Full' label is shown

ET-REQ-03unknown

Form collects: parent name, email, phone, child name, child age, preferred trial date (from available dates)

Acceptance Criteria

All fields are required; trial date picker shows only admin-defined trial-eligible dates with available capacity

ET-REQ-04unknown

If parent email matches an existing lead or parent record, pre-fill known fields

Acceptance Criteria

Parent name, email, phone are pre-filled; parent can edit pre-filled values

ET-REQ-05unknown

On submission, system creates or updates a lead record with status 'Trial Scheduled'

Acceptance Criteria

Lead record exists with correct status, child info, and selected trial date within 10 seconds of submission

ET-REQ-06unknown

On submission, parent receives a confirmation email with trial details and optional calendar invite

Acceptance Criteria

Email is sent within 2 minutes of submission; email contains program name, date, time, and location

ET-REQ-07unknown

Form collects: parent name, email, phone, child name, child age, optional coupon code

Acceptance Criteria

All fields except coupon are required; form validates child age against program age group

ET-REQ-08unknown

Price breakdown displays before submission: base fee, registration fee (or waived), coupon discount, total

Acceptance Criteria

Breakdown updates in real-time when coupon is applied; total matches calculated values

ET-REQ-09unknown

On submission, enrollment record is created with payment status 'pending'

Acceptance Criteria

Enrollment record exists with correct program, child, pricing, and payment_status = 'pending'

ET-REQ-10unknown

Each enrollment is for exactly one child; parents with multiple children submit separate enrollment forms

Acceptance Criteria

Form has fields for one child only; no multi-child selection is available

ET-REQ-11unknown

Confirmation screen shows: enrollment summary (program, child, price breakdown) and Zelle payment instructions (recipient info, amount, reference)

Acceptance Criteria

All enrollment details and payment instructions are visible; parent can screenshot or print the page

ET-REQ-12unknown

System sends a confirmation email with the same enrollment summary and payment instructions

Acceptance Criteria

Email is sent within 2 minutes; content matches confirmation screen
